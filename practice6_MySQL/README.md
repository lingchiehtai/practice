# MySQL 訂便當系統與資料分析

## 簡介
本專案旨在練習如何透過 MySQL 資料庫管理員工的訂餐資訊，並結合 Python (`pymysql`) 進行自動化操作與統計分析。專案模擬了企業內部統計單月餐點費用、熱門餐廳排行及跨表整合員工部門資訊的流程。

## 專案內容
專案包含以下檔案與組件：

- **資料檔案 (CSV)**:
  - `MySQL_order_lunch.csv`: 原始訂單紀錄，包含員工編號、餐廳、品項、價格與訂餐日期。
  - `MySQL_order_lunch_ID_name_department.csv`: 員工主檔，包含員工編號、姓名與所屬部門。
- **資料庫腳本 (SQL)**:
  - `MySQL_order_lunch語法.sql`: 包含建立資料表 (`TodayOrder`, `EmployeeList`)、跨表查詢 (JOIN) 與彙總分析 (GROUP BY) 的 SQL 指令。
- **自動化程式碼 (Jupyter Notebook)**:
  - `MySQL_order_lunch.ipynb`: 使用 Python 連結 MySQL 資料庫，執行資料表初始化、資料匯入與分析結果輸出的完整工作流。

## 重點分析
本專案實作了以下關鍵分析邏輯：

1. **單月餐費統計**:
   - 針對特定月份（如 6 月）計算每位員工的總餐費，方便財務核算。
2. **跨表資訊整合**:
   - 透過 `LEFT JOIN` 將訂單資料與員工主檔串接，產出包含員工姓名與部門的詳細餐費明細表。
3. **熱門程度排行**:
   - **熱門品項**: 統計各餐廳不同餐點的訂購次數，並進行降冪排序，分析員工口味偏好。
   - **熱門餐廳**: 統計各餐廳的總訂餐次數，作為後續訂餐決策的參考。
