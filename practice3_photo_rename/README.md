# Photo Rename with Keywords

## 簡介
用於批次處理照片檔案。透過 Google Gemini AI 分析照片內容和每日筆記，生成描述性關鍵字，並將其附加到檔案名稱中，提升檔案的可搜尋性和意義性。專案支援多組 API 金鑰自動切換，以處理配額限制。

## 專案內容
- **主要檔案**：`rename_photos_keywords.py` - 核心腳本，負責解析筆記、篩選檔案、呼叫 Gemini API 生成新檔名。
- **依賴**：
  - Python 3.x
  - `google-genai` (Google Gemini SDK)
  - 標準庫：`os`, `pathlib`, `re`, `time`
- **功能**：
  - 讀取 `mynote.txt` 筆記檔案，解析日期和內容。
  - 篩選未命名的照片檔案（格式如 `YYYY-MM-DD_NUM.ext`）。
  - 使用 Gemini AI 生成關鍵字，附加到原檔名。
  - 支援多 API 金鑰自動切換，處理 429/503 錯誤。
  - 統計重命名成功數量。

## 重點分析
- **筆記解析**：使用正則表達式提取日期（月/日格式）和對應內容，支援無筆記時直接分析圖片。
- **檔案篩選**：正則匹配未重命名檔案，避免重複處理已含關鍵字的檔案。
- **AI 整合**：上傳圖片到 Gemini，生成內容時結合筆記上下文。回應格式驗證確保新檔名正確。
- **錯誤處理**：重試邏輯處理 API 錯誤，自動切換金鑰，防止腳本中斷。


## 如何執行
1. **環境設定**：
   - 安裝依賴：`pip install google-genai`
   - 設定環境變數：將 Gemini API 金鑰設為 `GEMINI_API_KEY_1`、`GEMINI_API_KEY_2` 等（支援最多 6 組）。

2. **準備檔案**：
   - 將照片檔案放在專案目錄中，格式如 `2025-09-10_51.jpg`。
   - 建立 `mynote.txt`，內容格式如 `9/10 今日筆記內容`。

3. **運行腳本**：
   - 在終端機執行：`python rename_photos_keywords.py`
   - 腳本會自動處理檔案，輸出進度和統計。

注意：確保 API 金鑰有效，並監控配額使用。腳本在 Windows 上測試。
